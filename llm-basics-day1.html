DAY 01

① Transformer 架构 / Transformer Architecture

[LLM][架构][基础]

// 核心概念

Transformer 是 2017 年提出的完全基于 self-attention 的序列建模架构，彻底取代了 RNN/CNN 在 NLP 的主导地位。它重要的本质原因：将"信息的远距离传递"从 O(n) 的串行路径压缩到 O(1) 的并行可达——任意两个 token 之间一步可达，这是所有现代 LLM 智能的物理基础。

// 原理详解

一个标准 Transformer Block 由两个子层组成，每个子层包裹在残差 + LayerNorm 中：

  • Multi-Head Self-Attention：捕捉 token 之间的语义/句法关系
  • Position-wise FFN：两层 MLP（通常隐藏维度 = 4×d_model），完成非线性变换与"知识存储"
$\text{Block}(x) = \text{LN}(x + \text{FFN}(\text{LN}(x + \text{Attn}(x))))$

现代 LLM (GPT/Llama) 采用 decoder-only 结构 + Pre-LN（LayerNorm 前置，训练更稳）+ RMSNorm（去掉均值项，性能与 LayerNorm 接近但更快）+ SwiGLU 激活。一个 70B 模型典型配置：80 层、d_model=8192、64 个 head、FFN 维度 28672。

架构示意：

Input Tokens
   ↓
[Embedding + Positional Encoding]
   ↓
┌─ Block × N ──────────────┐
│  Self-Attention (causal) │  ← 残差
│  + LayerNorm             │
│  FFN (SwiGLU)            │  ← 残差
│  + LayerNorm             │
└──────────────────────────┘
   ↓
[LM Head: Linear → Softmax]
   ↓
Next Token Distribution

// 代码示例

import torch, torch.nn as nn

class TransformerBlock(nn.Module):
    def __init__(self, d=512, h=8, ff=2048):
        super().__init__()
        self.ln1 = nn.LayerNorm(d)
        self.attn = nn.MultiheadAttention(d, h, batch_first=True)
        self.ln2 = nn.LayerNorm(d)
        self.ffn = nn.Sequential(
            nn.Linear(d, ff), nn.GELU(), nn.Linear(ff, d))

    def forward(self, x, mask=None):
        # Pre-LN: 现代 LLM 标配，训练稳定性远优于 Post-LN
        h = self.ln1(x)
        x = x + self.attn(h, h, h, attn_mask=mask, need_weights=False)[0]
        x = x + self.ffn(self.ln2(x))
        return x

// 关键论文

  • Vaswani et al., 2017. Attention Is All You Need — 开山之作
  • Xiong et al., 2020. On Layer Normalization in the Transformer Architecture — Pre-LN vs Post-LN

// 跨学科联想

Transformer 的并行可达性，本质上与大脑皮层的全局工作空间理论 (Global Workspace Theory) 高度对应——意识的产生需要将局部信息"广播"到全脑。Attention 即是这种广播机制的数学表达。从复杂性科学看，Transformer 是一个全连接的动力系统，每个 token 的状态都由所有其他 token 共同决定，这与神经元集群的同步现象一致。

// 实践场景

经典应用：所有现代 LLM (GPT-4、Claude、Gemini、Llama) 的核心。
超级个体场景：理解 Transformer 让你能精准判断模型能力边界——例如知道为何 long-context 模型在中间位置会出现 "lost in the middle"（attention 在长序列上分配不均），从而设计更可靠的 RAG/Agent 工作流。

💭 思考题：如果 Transformer 的本质是"全连接信息广播"，那么人类大脑用稀疏的局部连接 + 远距离白质纤维束实现了类似功能。这种"稀疏 vs 稠密"的差异，是否暗示了下一代 LLM 架构的方向？

② Self-Attention 机制 / Self-Attention Mechanism

[LLM][核心][数学]

// 核心概念

Attention 是一个"软查表"操作：用 query 向量去匹配所有 key 向量，按相似度加权聚合 value。它让模型在每一层都能动态决定"看哪里"。重要性在于：它是 LLM 中唯一支持 token 间交互的算子（FFN 是 token-wise 独立的）。

// 原理详解

$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$

三个关键设计：

  • 缩放因子 $\sqrt{d_k}$：防止高维内积过大导致 softmax 饱和、梯度消失
  • Causal Mask：decoder-only LLM 中，将上三角设为 $-\infty$，保证 token $t$ 只看 $\le t$ 的位置
  • Multi-Head：将 d_model 拆成 h 份，每份独立做 attention，最后 concat。不同 head 学到不同类型的关系（句法、共指、远距离依赖）

现代变种：

  • GQA (Grouped-Query Attention)：多个 Q head 共享同一组 K/V head，显著降低 KV cache 显存（Llama-3、Mistral 采用）
  • MQA：极端版 GQA，所有 Q 共享 1 组 KV
  • FlashAttention：通过 tiling + 重计算，将 attention 的显存复杂度从 $O(n^2)$ 降到 $O(n)$，速度提升 2-4×

// 代码示例

import torch, torch.nn.functional as F

def scaled_dot_product_attention(Q, K, V, causal=True):
    # Q,K,V: (batch, heads, seq, d_k)
    d_k = Q.size(-1)
    scores = Q @ K.transpose(-2, -1) / d_k**0.5      # (B,H,S,S)
    if causal:
        mask = torch.triu(torch.ones_like(scores), diagonal=1).bool()
        scores = scores.masked_fill(mask, float('-inf'))
    attn = F.softmax(scores, dim=-1)
    return attn @ V                                   # (B,H,S,d_k)

# 生产环境直接用 PyTorch 2.0+ 的融合实现（自动调度 FlashAttention）
out = F.scaled_dot_product_attention(Q, K, V, is_causal=True)

// 关键论文

  • Dao et al., 2022. FlashAttention — IO-aware 的工程突破
  • Ainslie et al., 2023. GQA: Training Generalized Multi-Query Transformer

// 跨学科联想

Attention 的 softmax 权重分配与神经科学中的"选择性注意"惊人相似：丘脑-皮层回路通过抑制无关信号、放大相关信号实现注意聚焦。从量子力学类比：QK^T 内积像"波函数的内积"——衡量两个状态的相似度（重叠），softmax 像 Born 规则的概率化。从佛学视角：attention 即"作意"（manaskāra），是心识聚焦的最基础操作。

// 实践场景

经典应用：翻译、摘要、代码补全中的远距离依赖建模。
投资场景：在多 Agent 投资分析系统中，可以借鉴 attention 思想——让"决策 Agent"在多个"分析师 Agent"输出之间学习权重，而非简单平均，从而对高置信信号自动放大。

💭 思考题：Softmax 的"赢者通吃"倾向（temperature 1 时分布常常很尖）是 attention 的优点还是缺陷？人类思维中的"灵感跳跃"是否需要 attention 偶尔关注"低概率"位置？

③ Tokenization 分词 / Tokenization (BPE)

[LLM][预处理][工程]

// 核心概念

Tokenization 把原始字符串切分成模型词表中的离散单元 (token)。它是 LLM 的"感知器官"——决定了模型能看到什么、看不到什么。重要性常被低估：分词的边界决定了模型的算术能力、跨语言能力、甚至 prompt injection 的攻击面。

// 原理详解

主流方案是 BPE (Byte-Pair Encoding) 及其变种 BBPE (Byte-level BPE，GPT/Llama 采用)：

  1. 初始词表 = 所有 256 个字节
  2. 在大语料上统计相邻 token 对的频率，将最频繁的对合并为新 token
  3. 重复直到达到目标词表大小（GPT-4 约 10 万，Llama-3 约 12.8 万）

关键性质：

  • 无 OOV (out-of-vocabulary)：字节级 fallback 保证任何字符串可编码
  • 高频词为单 token，低频词被切碎；中文 token 效率远低于英文（约 2-3 倍 token 数）
  • 数字问题："12345" 可能被切成 "12" + "345"，导致算术不稳定。Llama-3 专门把数字按单字符 token 化
  • SentencePiece：BPE 的 unigram 变体，Llama/T5 使用，可端到端学 tokenizer

// 代码示例

from tiktoken import get_encoding

enc = get_encoding("cl100k_base")  # GPT-4 tokenizer
text = "Transformer changed everything. 注意力即一切。"
ids = enc.encode(text)
print(len(ids), ids[:8])
# 14 [9140, 64101, 5614, 4395, 13, 41642, 24816, 19483]
print([enc.decode([i]) for i in ids])
# ['Transform', 'er', ' changed', ' everything', '.', ' 注', '意', '力'...]

# 观察：'Transformer' 被切成 2 个 token，中文每字 1-2 个 token

// 关键论文

  • Sennrich et al., 2016. Neural Machine Translation of Rare Words with Subword Units — BPE 引入 NLP
  • Kudo & Richardson, 2018. SentencePiece

// 跨学科联想

Tokenization 类似认知科学中的"知觉单元化" (perceptual chunking)——大脑把连续输入切成有意义的块（音节、词、短语）。米勒的"7±2"工作记忆容量受 chunk 大小调制：chunk 越好，记忆越多。从复杂性科学看，BPE 的合并规则是自组织临界 (self-organized criticality)的体现：词表按 Zipf 分布涌现，与自然语言的统计规律一致。

// 实践场景

经典应用：所有 LLM 输入输出的预/后处理。
育儿/教育场景：理解 tokenization 后，与孩子讨论 ChatGPT 为何不会数 "strawberry" 里的 r——这是非常生动的"AI 局限性"教育素材，让孩子理解 AI 是有具体机制约束的、而非全知全能。

💭 思考题：如果未来 LLM 直接以"字节"为单位 (byte-level，无 BPE)，会得到什么？牺牲什么？Google 的 ByT5 / Mamba-byte 给出了部分答案——你认为这会让 AI 更"接近"人类的感知，还是更远离？

④ 位置编码 / Positional Encoding (RoPE)

[LLM][架构][长上下文]

// 核心概念

Self-Attention 本身是置换不变 (permutation invariant) 的——打乱输入顺序不改变输出。位置编码是注入"顺序信息"的唯一通道，决定了模型如何理解时间、因果、远近。重要性：它是 long-context 能力的核心瓶颈，几乎所有 100K+ context 模型的突破都来自位置编码的改进。

// 原理详解

三代演进：

  1. 正弦位置编码 (Sinusoidal, 2017)：原始 Transformer，固定函数 $PE_{(pos,2i)}=\sin(pos/10000^{2i/d})$，加到 embedding。问题：外推性差。
  2. Learned PE：直接学一个 $(\text{max\_len} \times d)$ 矩阵。问题：超过训练长度直接崩溃。
  3. RoPE (Rotary Position Embedding, 2021) ⭐ 现代主流：将 query/key 的每对维度看作复平面上的二维向量，按位置 $m$ 旋转角度 $m\theta_i$。
$q_m^T k_n = \text{Re}\left[\sum_i (q_m^{(i)} e^{im\theta_i})(k_n^{(i)} e^{in\theta_i})^*\right]$, 结果只依赖 $m-n$（相对位置！）

RoPE 的杀手锏：

  • 相对位置感知（内积只依赖距离）
  • 无需额外参数
  • 可通过 NTK scaling / YaRN 等技术外推到训练长度的 4-16 倍 → 这是 Llama-3 8K → 128K，Claude 200K context 的核心技术

// 代码示例

import torch

def rope(x, base=10000):
    # x: (B, H, S, D), D 必须是偶数
    B, H, S, D = x.shape
    half = D // 2
    freq = 1.0 / (base ** (torch.arange(0, half, device=x.device) / half))
    pos = torch.arange(S, device=x.device)
    angles = pos[:, None] * freq[None, :]            # (S, D/2)
    cos, sin = angles.cos(), angles.sin()
    x1, x2 = x[..., :half], x[..., half:]
    # 二维旋转：[x1, x2] -> [x1*cos - x2*sin, x1*sin + x2*cos]
    return torch.cat([x1*cos - x2*sin, x1*sin + x2*cos], dim=-1)

# 应用：对 Q 和 K 都做 rope，不对 V 做
Q_rot = rope(Q); K_rot = rope(K)

// 关键论文

  • Su et al., 2021. RoFormer: Enhanced Transformer with Rotary Position Embedding
  • Peng et al., 2023. YaRN: Efficient Context Window Extension

// 跨学科联想

RoPE 用"相位旋转"编码位置，与量子力学中的相位编码本质相同——量子计算的相位干涉、傅立叶变换、波函数演化都用相同的数学结构 $e^{i\theta}$。从神经科学看，海马体的"theta 相位编码"（神经元放电相对于 theta 波的相位编码位置/时间）几乎是 RoPE 的生物对应物。这种巧合提示：相位是编码"序"最自然的方式。

// 实践场景

经典应用：长文档问答、代码仓库级理解、视频时序建模。
超级个体场景：构建"个人知识库 Agent"时，理解 RoPE + YaRN 让你能判断给定模型在 long context 上的真实能力——例如某些模型号称 1M context，但在 100K 之后 retrieval 准确率骤降。可指导你选择"长上下文一次性吞入"还是"RAG 分块检索"的架构。

💭 思考题：人脑没有"位置编码"模块，却能处理任意长度的故事。这是因为我们的"上下文"被压缩到了 episodic memory（事件记忆），而非保留所有原始 token。下一代 AI 的"长上下文"，是该继续暴力扩大 window，还是该学习压缩-检索？
